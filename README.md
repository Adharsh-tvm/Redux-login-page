# 🔐 MERN Auth & Admin Dashboard with Redux

A full-stack MERN (MongoDB, Express, React, Node.js) application featuring secure role-based authentication, user profile management, and an administrative dashboard. This project heavily utilizes **Redux Toolkit** for global state management and **Material-UI (MUI)** for a polished, responsive user interface.



## ✨ Core Features

* **Role-Based Access Control (RBAC):** Distinct routing and permissions for standard Users and Admins.
* **Authentication System:** Secure registration, login, and logout flow utilizing JSON Web Tokens (JWT) stored safely in HTTP-only cookies.
* **Global State Management:** Centralized client-side state handling using Redux Toolkit slices.
* **Profile Management:** Users can update their credentials and upload profile pictures directly to cloud storage.
* **Admin Dashboard:** Admins have full CRUD (Create, Read, Update, Delete) capabilities over user accounts.
* **Cloud Media Storage:** Direct integration with Cloudinary for handling and serving profile picture uploads.

* **File Uploads:** Multer, Cloudinary (`multer-storage-cloudinary`)

## 🏛️ Architecture & Folder Structure

The application separates the client and server into two distinct directories, ensuring clean separation of concerns.

### **Frontend Architecture**
The frontend follows a feature-based structure heavily reliant on Redux for data flow. 



* **`/redux`:** Contains the global `store.js`, async `actions/` (e.g., login dispatches), and `slices/` (e.g., `authSlice.js` which manages user state and syncs with `localStorage`).
* **`/pages`:** View components for individual routes (`Home`, `Login`, `Admin`, `Profile`).
* **`/services`:** Axios API wrappers (`authService.js`) to keep components clean of raw HTTP logic.

### **Backend Architecture**
The backend follows a standard MVC-inspired architectural pattern:
* **`/routes`:** Defines API endpoints and attaches appropriate middleware (e.g., `userRoutes.js`).
* **`/controllers`:** Houses the core business logic (e.g., `createUser`, `loginUser`, `updateUserProfilePic`).
* **`/models`:** Mongoose schemas defining the data structure (e.g., `userModel.js`).
* **`/middlewares`:** Custom interceptors for error handling (`asyncHandler.js`), JWT verification (`authenticate`), and role checking (`authorizeAdmin`).

## 🔒 Security Measures



This project implements several critical security practices to protect user data:

* **HTTP-Only Cookies:** JWTs are intentionally *not* stored in `localStorage`. Instead, they are generated by the server and sent as `HttpOnly`, `Secure`, `SameSite=Strict` cookies to mitigate Cross-Site Scripting (XSS) attacks.
* **Password Hashing:** Plain-text passwords are never saved. They are salted and hashed using `bcryptjs` before database insertion.
* **Protected Routes (Backend):** The `authenticate` middleware verifies the JWT signature on sensitive API routes. The `authorizeAdmin` middleware further restricts access to admin-only endpoints.
* **Protected Routes (Frontend):** React Router leverages Redux state and local storage to redirect unauthorized users away from protected views.
* **Input Validation:** Client-side form validation is enforced using `react-hook-form` to prevent malformed API requests.
* **CORS Policy:** Cross-Origin Resource Sharing is strictly configured to only accept requests (with credentials) from the Vite frontend port (`http://localhost:3000`).

## 🚀 Getting Started

### Prerequisites
* Node.js installed
* MongoDB connection string (local or MongoDB Atlas)
* Cloudinary Account (for image uploads)
* **pnpm** installed (`npm i -g pnpm`)

Create a `.env` file in the `serverside/` directory and add the following keys:
